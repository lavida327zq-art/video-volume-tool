# 视频音量检测工具

基于 Python + FFmpeg 的视频/音频音量检测工具，支持**本地视频上传**、**抖音链接解析**和**命令行批量扫描**，结果统一以 **0 dBFS（数字满量程）** 为基准，方便横向比较不同视频的音量水平。

---

## 功能特性

| 功能 | 说明 |
|------|------|
| 双输入方式 | 支持拖拽上传本地视频，或粘贴抖音视频链接 |
| 抖音在线解析 | 通过 API 解析抖音分享链接，无需下载即可分析音量 |
| 双指标输出 | 平均音量（RMS）+ 最大音量（峰值） |
| 统一基准 | 0 dBFS 为数字音频最大值，所有结果基于同一基准 |
| 状态高亮 | 自动标记「正常」「过低」「无音轨」「削波/失败」 |
| 结果导出 | 支持 CSV、JSON 两种格式下载 |
| 历史记录 | 每次检测结果自动保存，可回看、重载、重新导出 |
| 资源点提示 | 使用抖音解析 API 后，右下角提示本次消耗与剩余资源点 |

## 限制条件

### 硬件与性能限制

| 维度 | 建议值 | 说明 |
|------|--------|------|
| **单次上传数量** | 20-30 个以内 | 无硬性上限，但越多越慢 |
| **单个文件大小** | 100MB 以内 | 默认 Streamlit 限制 200MB，可通过 `--server.maxUploadSize 1024` 调至 1GB |
| **内存占用** | 200MB 视频 × 20 ≈ 4GB | 上传文件会暂存内存，超过可用内存会卡顿 |
| **处理速度** | 串行处理，每个视频几秒到几十秒 | 取决于视频长度和 CPU |
| **网络要求** | 必须能访问公网 | 抖音解析、视频下载均需外网 |

### 依赖要求

- Python 3.8+
- FFmpeg 4.0+（必须先安装）
- Streamlit、Pandas

## 使用方法

### 1. 安装依赖

```bash
# 1. 安装 FFmpeg
brew install ffmpeg  # macOS
# 或从 https://ffmpeg.org/download.html 下载

# 2. 安装 Python 库
pip install streamlit pandas
```

---

### 2. Web 界面（推荐）

启动服务（本机使用）：

```bash
python3 -m streamlit run volume_web.py --server.port 8501 --server.headless true
```

启动服务（允许内网同事访问）：

```bash
python3 -m streamlit run volume_web.py --server.port 8501 --server.headless true --server.address 0.0.0.0
```

> `--server.address 0.0.0.0` 让服务监听所有网卡，同内网的人才能通过你的 IP 访问。

使用 macOS 自带终端（Terminal.app）启动服务：

```bash
cd /Users/shaozi/Documents/企业Trae/视频转文稿 && python3 -m streamlit run volume_web.py --server.port 8501 --server.headless true --server.address 0.0.0.0
```

> 在 macOS 终端中需先 `cd` 到项目目录，否则会报 `File does not exist: volume_web.py`。

访问：
- 本机：http://localhost:8501
- 同局域网同事：http://你的内网IP:8501（如 http://192.168.1.100:8501）

关闭服务：

```bash
# 方式 1：在运行服务的终端窗口按 Ctrl + C
# 方式 2：在任意终端执行命令杀掉进程
pkill -f "streamlit run volume_web.py"
```

**为什么你的电脑需要开机**：Streamlit 服务运行在你的电脑上，服务关闭后网页无法访问。同事访问的是你的电脑提供的服务，因此你必须保持电脑开机且服务运行。

#### 服务运行状态说明

| 场景 | 服务是否关闭 | 说明 |
|------|-------------|------|
| 电脑锁屏 | ❌ 不关闭 | 锁屏只锁定界面，后台进程正常运行 |
| 电脑睡眠/合盖 | ⚠️ 暂停 | 进程不终止但暂停运行，网络断开，唤醒后恢复。睡眠期间他人无法访问 |
| 退出 Trae 应用 | ✅ 会关闭 | 服务运行在 Trae 终端中，退出 Trae 会终止终端会话及其中进程 |
| 关闭运行服务的终端标签 | ✅ 会关闭 | 终端被关闭，进程随之终止 |
| 网络断开 | ❌ 不关闭 | 服务正常运行，但他人无法访问 |
| macOS 内存不足 | ⚠️ 可能被杀 | 系统可能强制终止占内存的进程 |
| 系统更新重启 | ✅ 会关闭 | 需重新启动服务 |

> **提示**：如果想脱离 Trae 独立运行服务，可在 macOS 自带终端（Terminal.app）中执行同样的启动命令，这样关闭 Trae 也不影响服务。

操作流程：
1. 选择输入方式：拖拽上传本地视频文件，或在文本框中粘贴抖音视频链接（每行一个）
2. 设置音量阈值（默认 **-25 dBFS**，可根据业务调整）
3. 点击「开始检测」
4. 查看结果并下载 CSV/JSON 报告
5. 在「📜 历史记录」面板中回看或重新导出旧结果

> **CSV 导出注意**：若作者名/视频名以 `=`、`+`、`-`、`@` 开头，导出时会自动加单引号转义，避免 Excel 误识别为公式。

### 3. 命令行工具

适用于大批量/自动化场景。

```bash
# 检测单个文件
python3 detect_volume.py path/to/video.mp4

# 扫描整个目录
python3 detect_volume.py path/to/directory

# 设置阈值并导出 JSON
python3 detect_volume.py path/to/directory --threshold -30 --json report.json
```

## 抖音链接检测说明

1. 在 Web 页面文本框中粘贴抖音视频链接，例如：
   ```
   https://www.douyin.com/video/7646284426151141541
   ```
2. 工具会自动调用 cyanlis 抖音详情 v4 接口解析链接，返回无水印视频地址
3. 工具将视频临时下载到本地，使用 FFmpeg 分析音量，完成后自动清理临时文件
4. 结果表格的「来源」列会显示为可点击的超链接，跳转至原视频页面

## 指标说明

| 指标 | 全称 | 说明 |
|------|------|------|
| **平均音量 (mean_volume)** | RMS (Root Mean Square) | 反映整体响度，用于判断「音量过低」 |
| **最大音量 (max_volume)** | Peak Value | 峰值，若为 0.00 表示已削波（音量过大导致失真） |

### 阈值参考（dBFS）

> 默认阈值为 **-25 dBFS**，基于人工标注样本校准，适合口播/轻声/孕期陪伴类内容。

| 范围 | 状态 | 建议 |
|------|------|------|
| `> -25` | ✅ 正常 | 音量合适 |
| `-25 ~ -16` | ⚠️ 偏轻柔 | 可接受，无需重录 |
| `< -25` | ❌ 过低 | 建议重新录制或提高音量 |

### 关于 dBFS

- 0 dBFS = 数字音频最大上限，样本值已达 ±32767（16位）
- 所有值 ≤ 0
- 越接近 0 → 音量越大
- 越远离 0（绝对值越大）→ 音量越小

## 历史记录

每次检测完成后，结果会自动保存到 `.volume_history/` 目录中。

| 配置项 | 说明 |
|--------|------|
| 保存位置 | 与 `volume_web.py` 同级的隐藏目录 `.volume_history/` |
| 保留数量 | 最多保留最近 **20** 条记录 |
| 过期清理 | 自动清理超过 **7** 天的记录 |
| 支持操作 | 查看、下载 CSV、下载 JSON、删除单条、清空全部 |
